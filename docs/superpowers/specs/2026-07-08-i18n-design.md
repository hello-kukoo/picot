# Picot 国际化 (i18n) 设计文档

> **日期**: 2026-07-08
> **状态**: 设计已批准，待实施
> **作者**: Dr. Lin & Mr. Spock

## 1. 目标

为 Picot 添加国际化支持，首期支持**中文**和**英文**，架构上预留扩展其他语言的能力。

**核心需求：**
1. 所有页面的用户可见文本支持 i18n
2. Settings → General 新增"Language"设置，允许用户手动选择：System Default / 中文 / English
3. 语言偏好跨 workspace 窗口共享（每个 workspace 运行在不同 `localhost:port`）

## 2. 约束

| 约束 | 说明 |
|------|------|
| 纯原生 JS，无框架 | i18n 引擎必须零依赖、零构建步骤 |
| 无前端构建管线 | ES modules 直接以 `<script type=module>` 加载，不能引入打包工具 |
| Cookie 持久化（非 localStorage） | 每个 workspace 是不同端口的 `localhost:300X`，localStorage 按端口隔离。语言偏好必须跨端口共享——与 `themes.js` 完全同样的约束和模式 |
| Tauri v2 桌面应用 | 非 Web 部署，无 SSR |
| `cost.html` 在 iframe 中加载 | 同源同端口，共享 cookie，无需特殊处理 |
| `bootstrap.html` 是独立错误页 | 纯静态 `<script>`，无 ES module，需特殊处理 |

## 3. 方案选择

**选定方案：自建轻量 i18n 模块**（放弃 i18next / 多份 HTML 预渲染方案）

**理由：**
- Picot 的 i18n 需求极简：~500 个扁平字符串 + `{var}` 插值，不需要复数、延迟加载、命名空间等高级功能
- 自建引擎 ~150 行代码，与 `themes.js` 的 cookie 模式完全对称
- 零依赖、零构建，符合项目核心约束
- 后续加语言只需加一个 JSON 文件

## 4. 架构设计

### 4.1 文件结构

```
public/
  i18n.js                      ← 新模块：i18n 引擎（核心）
  locales/
    en.json                    ← 英文翻译（source of truth）
    zh.json                    ← 中文翻译
  index.html                   ← 静态文本加 data-i18n 属性
  cost.html                    ← 静态文本加 data-i18n 属性（Phase 2）
  bootstrap.html               ← 内联翻译逻辑（Phase 2）
  app.js                       ← 引入 i18n，调用 initI18n()，动态文本改 t() 调用
  各模块 .js                    ← 动态文本改 t() 调用
  app-voice-input.js           ← recognition.lang 跟随 locale
```

### 4.2 i18n 引擎设计 (`public/i18n.js`)

#### 导出 API

```javascript
// 初始化：读取 cookie / navigator.language，加载对应 locale JSON，扫描 DOM
export async function initI18n(): Promise<void>;

// 翻译函数：支持 {var} 插值
export function t(key: string, params?: Record<string, string|number>): string;

// 设置语言：写 cookie，重新加载 locale，重新扫描 DOM
export async function setLocale(locale: string): Promise<void>;

// 获取当前有效语言代码（"en" | "zh"）
export function getLocale(): string;

// 获取用户选择（"system" | "en" | "zh"），用于设置面板高亮
export function getLanguagePreference(): string;

// 语言选项常量（设置面板用）
export const LANGUAGES: Array<{ value: string; label: string }>;
```

#### 核心逻辑

```
initI18n()
  ├── preference = readCookie("picot-language") ?? "system"
  ├── locale = resolveLocale(preference)
  │     ├── preference === "system" → navigator.language 前缀匹配
  │     │     "zh*" → "zh", 其余 → "en"
  │     └── preference === "en" | "zh" → 直接使用
  ├── messages = await fetch(`/locales/${locale}.json`)
  ├── document.documentElement.lang = locale
  └── applyTranslations()

applyTranslations()
  ├── 扫描 [data-i18n]        → element.textContent = t(key)
  ├── 扫描 [data-i18n-ph]     → element.placeholder = t(key)
  ├── 扫描 [data-i18n-title]  → element.title = t(key)
  ├── 扫描 [data-i18n-html]   → element.innerHTML = t(key)（需 escape 的场景不用此属性）
  └── 扫描 [data-i18n-aria-label] → element.setAttribute("aria-label", t(key))

setLocale(preference)
  ├── writeCookie("picot-language", preference)
  ├── locale = resolveLocale(preference)
  ├── messages = await fetch(`/locales/${locale}.json`)
  └── applyTranslations()  ← 同一页面内即时切换，无需刷新

t(key, params)
  ├── 从 messages 嵌套对象中按 "." 分隔路径查找
  ├── 找到 → 替换 {var} 占位符 → 返回
  ├── 未找到 → 尝试英文 fallback → 返回
  └── 英文也没有 → 返回 key 本身（开发期 console.warn）
```

#### Cookie 策略

```javascript
const LANG_COOKIE = "picot-language";
const LANG_COOKIE_MAX_AGE = 60 * 60 * 24 * 365 * 10; // 10 年

// 值为 "system" | "en" | "zh"
// 写入：document.cookie = `${LANG_COOKIE}=${value}; max-age=...; path=/`;
// 读取：与 themes.js 的 readThemeCookie() 相同的正则方式
```

> **注意**：cookie 值存的是用户偏好（`"system"` / `"en"` / `"zh"`），而非解析后的 locale。这样当系统语言改变时，"System Default" 用户会自动跟随。

### 4.3 Key 命名规范

- **点分嵌套**，按模块/功能区域分组：`sidebar.settings`、`composer.placeholder`
- 第一级对应模块名（`sidebar`、`composer`、`settings`、`files`、`status`、`messages`）
- 避免超过 3 层嵌套
- 唯一性：同一字符串在不同上下文中可能需要不同翻译（如 "Close" 作为动作 vs 状态），用上下文前缀区分

### 4.4 HTML 标注约定

| 属性 | 替换目标 | 示例 |
|------|---------|------|
| `data-i18n="key"` | `textContent` | `<span data-i18n="sidebar.settings">Settings</span>` |
| `data-i18n-ph="key"` | `placeholder` | `<input data-i18n-ph="sidebar.search" placeholder="Search...">` |
| `data-i18n-title="key"` | `title` | `<button data-i18n-title="sidebar.openFolder" title="Open folder">` |
| `data-i18n-aria-label="key"` | `aria-label` | `<button data-i18n-aria-label="sidebar.refresh" aria-label="Refresh">` |
| `data-i18n-html="key"` | `innerHTML`（仅安全文本） | `<p data-i18n-html="settings.configHelp">...</p>` |

**原则**：HTML 中保留英文原文作为 fallback（在 i18n 初始化前的极短时间内可见），避免空文本闪烁。

### 4.5 语言选择器 UI

在 Settings → General → Appearance 下方新增 **Language** section：

```html
<div class="settings-section">
  <div class="settings-section-title" data-i18n="settings.language.title">Language</div>
  <div class="settings-language-options" id="settings-language-options">
    <!-- 由 JS 动态生成 -->
  </div>
  <p class="settings-help" data-i18n="settings.language.description">
    Choose the interface language. Changes apply immediately.
  </p>
</div>
```

渲染逻辑（参考 `buildThemeGrid()` 的模式）：

```javascript
const LANGUAGES = [
  { value: "system", label: "System default", labelZh: "跟随系统" },
  { value: "en", label: "English" },
  { value: "zh", label: "中文" },
];

// 渲染为 radio 按钮组或 segmented control
function buildLanguageSelector() {
  const container = document.getElementById("settings-language-options");
  const current = getLanguagePreference();
  container.innerHTML = "";
  for (const lang of LANGUAGES) {
    const label = document.createElement("label");
    label.className = "settings-language-option";
    const radio = document.createElement("input");
    radio.type = "radio";
    radio.name = "language";
    radio.value = lang.value;
    radio.checked = current === lang.value;
    radio.addEventListener("change", () => setLocale(lang.value));
    const span = document.createElement("span");
    span.textContent = lang.label; // 语言名称始终用本语种显示，不翻译
    label.appendChild(radio);
    label.appendChild(span);
    container.appendChild(label);
  }
}
```

**设计决策**：语言名称始终用本语种显示（"English"、"中文"），不翻译——用户需要认出自己要选的语言。

### 4.6 初始化时序

`app.js` 中的初始化顺序，`initI18n()` 放在 `applyTheme()` 旁边：

```javascript
// app.js 初始化区域（约 line 3552）

// Restore saved theme
const savedTheme = getCurrentTheme();
applyTheme(savedTheme);

// Initialize i18n (async, but 不阻塞 — DOM 标注保留英文 fallback)
initI18n().then(() => {
  // 初始化完成后重新渲染动态生成的 UI 文本
  buildLanguageSelector();
});
```

**关于异步加载**：`initI18n()` 需 `fetch` locale JSON，是异步的。但：
1. HTML 中的英文原文在 i18n 初始化前可见（作为 fallback）
2. locale JSON 很小（~10–15KB），加载在毫秒级
3. 对于中文用户，会有极短暂的英文→中文闪烁，可接受（与 theme 闪烁同理）

**消除闪烁的优化**（可选）：在 `<head>` 中加一段同步 `<script>`，在 CSS/HTML 解析前读 cookie 并设 `document.documentElement.lang`，但 locale JSON 仍需异步加载。鉴于闪烁极短，Phase 1 不做此优化。

### 4.7 动态文本改造模式

模块中的 `innerHTML` / `textContent` 硬编码字符串改为 `t()` 调用：

```javascript
// 改造前
this.container.innerHTML = '<div class="file-loading">Loading…</div>';

// 改造后
import { t } from "./i18n.js";
this.container.innerHTML = `<div class="file-loading">${t("files.loading")}</div>`;
```

**带插值的改造**：

```javascript
// 改造前
statusText.textContent = `Exported: ${data.data.path}`;

// 改造后
statusText.textContent = t("status.exported", { path: data.data.path });
// en.json: "status": { "exported": "Exported: {path}" }
// zh.json: "status": { "exported": "已导出：{path}" }
```

### 4.9 语音输入跟随语言

```javascript
// app-voice-input.js 改造
import { getLocale } from "./i18n.js";

// 改造前：recognition.lang = "en-AU";
// 改造后：
const locale = getLocale();
recognition.lang = locale === "zh" ? "zh-CN" : "en-AU";
```

### 4.10 Cost 页面（iframe）i18n

`cost.html` 在 iframe 中加载，同源同端口，共享 cookie。`cost.js` 初始化时调用 `initI18n()` 即可，与主页完全相同的机制。

```javascript
// cost.js 顶部
import { initI18n } from "./i18n.js";

// 在 syncThemeFromParent() 旁加
await initI18n();
```

### 4.11 Bootstrap 错误页 i18n

`bootstrap.html` 是独立错误页，使用内联 `<script>`（非 ES module），只有约 8 个字符串。两种方案：

**方案 A（推荐）**：读取 cookie，用内联 JS 替换文本。简单的 `if (lang === "zh")` 分支即可，不值得引入完整 i18n 引擎。

**方案 B**：跳过 i18n，保持英文。错误页出现频率极低，英文可接受。

→ **Phase 2 再决定**。

## 6. 测试策略

### 单元测试 (`public/i18n.test.js`)

| 测试用例 | 说明 |
|---------|------|
| `t()` 基本翻译 | key 存在时返回正确翻译 |
| `t()` 插值 | `{var}` 占位符正确替换 |
| `t()` missing key | 未找到 key 时返回 key 本身 + console.warn |
| `t()` 英文 fallback | zh.json 缺某 key 时 fallback 到 en |
| `resolveLocale("system")` + `navigator.language="zh-CN"` | → "zh" |
| `resolveLocale("system")` + `navigator.language="en-US"` | → "en" |
| `resolveLocale("zh")` | → "zh" |
| `getLanguagePreference()` 无 cookie | → "system" |
| `setLocale("zh")` 写 cookie | cookie 值为 "zh" |
| `applyTranslations()` DOM 扫描 | `data-i18n` 元素文本被替换 |

### 集成验证

1. **语言切换即时生效**：在设置中切换语言，整个 UI（静态 + 动态文本）立即变化，无需刷新
2. **跨窗口一致性**：在 workspace A 切换语言，打开 workspace B 窗口，B 自动使用相同语言
3. **System Default**：设置为 "System Default"，修改 OS 语言后重新打开窗口，语言跟随 OS
4. **cost 仪表盘**（Phase 2）：在 iframe 中的 cost 页面也跟随语言设置

## 7. 风险与缓解

| 风险 | 概率 | 缓解 |
|------|------|------|
| 翻译遗漏（某 key 只在 en.json 没有 zh.json） | 高 | `t()` 自动 fallback 到英文；CI 可加脚本检查 key 完整性 |
| 动态文本遗漏 i18n 改造 | 中 | 逐模块 checklist，code review 对照 |
| 翻译质量 | 中 | 中文翻译由 Dr. Lin 审校 |
| `app.js` 122KB 改动冲突 | 中 | i18n 改动以 `t()` 替换为主，改动点小而散，降低合并冲突 |
| locale JSON 加载延迟导致闪烁 | 低 | HTML 保留英文 fallback；JSON < 15KB，毫秒级加载 |

## 8. 非目标 (Non-Goals)

- **不做** RTL（从右到左）语言支持
- **不做** 日期/数字本地化格式（Picot 不显示这类内容）
- **不做** 自动翻译（所有翻译人工维护）
- **不做** 按需加载 locale 文件（两种语言总共 ~30KB，不值得懒加载）
- **不做** 服务端翻译（pi 的 RPC 响应不翻译——那是 pi 的职责，Picot 只管自己的 UI）
